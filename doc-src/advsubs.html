From the examples in the previous section, it may seem that text between the @!...!@ symbols is treated simply as an identifier to be replaced. This is not the case - CubicTemp actually evaluates everything between the @ symbols as an expression in the namespace specified by the keyword arguments. It is the result of this evaluation that is converted to a string with the built-in function repr, and placed in the template. This means that any valid Python expression which can be evaluated in the indicated namespace can appear in an @ tag.

To illustrate this, take a look at the template below:

@!foo!@ times two is @!foo*2!@
@!foo!@ squared is @!foo*foo!@

...and this accompanying code snippet:

@!colourPython(code)!@

The output here would look like this:

3 times two is 6
3 squared is 9

Similarly, if "foo" happened to be a dictionary, we could extract a value from it inside a template like this:

@!foo["bar"]!@

If "foo" was an object with an attribute "x", we could reference the attribute by saying:

@!foo.x!@
